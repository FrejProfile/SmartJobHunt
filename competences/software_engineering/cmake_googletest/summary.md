CMake build pipeline experience across multiple project types:
- Embedded C++ projects with unit testing integration

Managing dependencies, build targets and test integration across

for embedded code, integrated into CMake build pipeline.

**pytest** — Python testing, applied in data processing and algorithm
validation projects including the Amfitech work.

Both frameworks used in real projects, not just coursework exercises.

## ROS2 and Gazebo Context
Extensive CMake usage through ROS2 development and Gazebo simulation
projects. ROS2 enforces structured CMake practices — package dependencies,
message generation, launch files — giving broader CMake exposure beyond
simple embedded builds.

## Honest Framing
Smaller scale than a production CI/CD pipeline — do not claim enterprise

structured build and test practices across multiple frameworks and
project types including embedded, simulation and algorithm work.
